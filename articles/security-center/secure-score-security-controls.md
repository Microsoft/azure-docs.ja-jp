# <a name="the-enhanced-secure-score-preview"></a>強化されたセキュア スコア (プレビュー) 

この記事では、強化されたセキュア スコア (現在はプレビュー段階)、付随するセキュリティ コントロール、およびその利点について説明します。 また、スコアの計算方法についても説明します。

## <a name="introduction-to-secure-score"></a>セキュア スコアの概要

Azure Security Center には、現在のセキュリティ状況を把握すること、およびセキュリティを効率的かつ効果的に向上させるという 2 つの主な目標があります。 これらの目標を達成できるようにする Security Center の中心となる要素が、セキュア スコアです。

Security Center は、セキュリティの問題について、リソース、サブスクリプション、および組織を継続的に評価します。 その後、すべての結果を 1 つのスコアに集約して、現在のセキュリティの状況を一目で確認できるようにします。スコアが高くなるほど、識別されたリスク レベルが低下します。 スコアを使用して、組織内のセキュリティの取り組みとプロジェクトを追跡します。 

*強化された*セキュア スコア (現時点ではプレビュー段階) は、**攻撃面にフォーカス**しているため、次の 3 つの利点があります。

- **セキュリティ コントロール** - セキュリティに関する推奨事項が、脆弱な攻撃面をより適切に示す論理セットにグループ化されるようになりました。 詳細については、「[セキュア スコアの計算方法](secure-score-security-controls.md#how-the-secure-score-is-calculated)」を参照してください。

- **全体的な体制をより適切に反映する総合スコア** - ポイントは、推奨レベルで与えられていました。 この拡張機能を使用すると、コントロール内の 1 つのリソースに関する推奨事項を*すべて*修復した場合にのみ、スコアが向上します。 これは、リソースのセキュリティが向上した場合にのみスコアが向上することを意味します。 

- **見やすくなった個々の攻撃面のセキュリティ状態** - [セキュア スコア] ページでセキュリティ コントロールごとのスコアを表示することで、組織が個々の攻撃面をどの程度セキュリティで保護できているかを詳細に表示できるようになりました。

次のスクリーンショットに示すように、強化されたセキュア スコアはパーセンテージとして表示されます。

[![強化されたセキュア スコア (プレビュー) にパーセンテージが含まれるようになりました](media/secure-score-security-controls/secure-score-with-percentage.png)](media/secure-score-security-controls/secure-score-with-percentage.png#lightbox)


## <a name="locating-your-secure-score"></a>セキュア スコアの検索

Security Center には、スコアが目立つように表示されます。スコアは、[概要] ページの最初に表示されます。 専用のセキュア スコア ページをクリックすると、スコアがサブスクリプション別に分類されていることがわかります。 サブスクリプションを 1 つクリックすると、優先度が設定された推奨事項と、修復によるスコアへの影響を示す詳細な一覧が表示されます。

## <a name="how-the-secure-score-is-calculated"></a>セキュア スコアの計算方法 

このプレビューの前に、Security Center は各推奨事項を個別に検討し、重大度に基づいて値を割り当てました。 これまで、セキュリティ体制の向上に取り組むセキュリティ チームは、すべての結果一覧に基づいて Security Center 推奨事項への応答に優先順位を付ける必要がありました。 セキュア スコアは、1 つのリソースの推奨事項を修復するたびに向上していました。

セキュア スコアの強化の一環として、推奨事項が**セキュリティ コントロール**にグループ化されました。 コントロールには、これらのレコメンデーションを有効にするための一連のセキュリティに関する推奨事項と指示内容があります。 コントロールは、関連する推奨事項を論理的にグループ化したものです。 ポイントは推奨レベルでは付与されなくなりました。 その代わりに、コントロール内の 1 つのリソースに関する推奨事項を*すべて*修復した場合にのみ、スコアが向上します。

総合的なセキュア スコアに対する各セキュリティ コントロールの影響は、[推奨事項] ページで明確に示されています。

[![強化されたセキュア スコア (プレビュー) にセキュリティ コントロールが導入されました](media/secure-score-security-controls/security-controls.png)](media/secure-score-security-controls/security-controls.png#lightbox)

セキュリティ コントロールで取得可能なポイントをすべて取得するには、すべてのリソースがセキュリティ コントロール内のすべてのセキュリティの推奨事項に準拠している必要があります。 たとえば、Security Center には、管理ポートをセキュリティで保護する方法に関する推奨事項が複数あります。 以前は、これらの関連する推奨事項の一部を修復して一部を未解決のまま残していても、セキュア スコアを上げることができました。 客観的に見れば、すべてを解決するまでセキュリティが改善されないことは明白です。 今後は、セキュア スコアを改善するには、すべての推薦事項を修復する必要があります。

たとえば、"システム更新プログラムの適用" というセキュリティ コントロールのスコアは最大 6 ポイントです。これは、コントロールの潜在的な増加値に関するヒントに表示されます。

[![セキュリティ コントロール "システム更新プログラムの適用"](media/secure-score-security-controls/apply-system-updates-control.png)](media/secure-score-security-controls/apply-system-updates-control.png#lightbox)

上のスクリーンショットのセキュリティ コントロール "システム更新プログラムの適用" の潜在的な増加値は、"2% (1 ポイント)" と表示されています。 これは、このコントロールのすべての推奨事項を修復すると、スコアが 2% (この場合は 1 ポイント) 増加することを意味します。 簡潔にするため、推奨事項リストの [Potential increase]\(潜在的な増加\) 列の値は整数に丸められます。 ツールヒントには、次の正確な値が表示されます。

* **Potential increase (潜在的な増加)** - コントロール内で取得できる残りのポイント。 これらのポイントをセキュア スコアに追加するには、コントロールのすべての推奨事項を修復します。 上の例では、コントロールに対して表示されている 1 ポイントは、実際には 0.96 ポイントです。
* **Current score (現在のスコア)** - このコントロールの現在のスコアです。 合計スコアは、各コントロールを基にしています。 この例のコントロールは、5.04 ポイントを合計に貢献しています。 
* **Max score (最大スコア)** - 前の 2 つの値の合計です。

## <a name="improving-your-secure-score"></a>セキュア スコアを向上する

セキュア スコアを向上させるには、推奨事項リストのセキュリティの推奨事項を修復してください。 各推奨事項は、リソースごとに手動で修復するか、リソース グループに推奨設定の修復を迅速に適用するために**クイック修正** オプション (利用可能な場合) を使用して修復することもできます。 詳細については、「[推奨事項の修復](security-center-remediate-recommendations.md)」を参照してください。

セキュア スコアに影響するのは、組み込みの推奨事項のみです。

## <a name="secure-score-faq"></a>セキュア スコアに関する FAQ

### <a name="why-has-my-secure-score-gone-down"></a>セキュア スコアが下がったのはなぜですか。
この強化されたセキュア スコアで導入された変更により、リソースでポイントを受信するには、すべての推奨事項を解決する必要があります。 さらに、スコアのスケールが 0-10 に変更されました。

### <a name="if-i-address-only-three-out-of-four-recommendations-in-a-security-control-will-my-secure-score-change"></a>セキュリティ コントロールの 4 つの推奨事項のうち 3 つだけに対処した場合、セキュア スコアは変わりますか。
いいえ。1 つのリソースのすべての推奨事項を修復するまでスコアは変更されません。 コントロールの最大スコアを取得するには、すべてのリソースのすべての推奨事項を修復する必要があります。

### <a name="will-this-enhanced-secure-score-replace-the-existing-secure-score"></a>既存のセキュア スコアは、強化されたセキュア スコアに置き換えられますか。 
はい。ただし、移行を容易にするため、しばらくの間は併用されます。

### <a name="if-a-recommendation-isnt-applicable-to-me-and-i-disable-it-in-the-policy-will-my-security-control-be-fulfilled-and-my-secure-score-updated"></a>推奨事項が自分に該当しないためポリシーで無効にした場合、セキュリティ コントロールは達成され、セキュア スコアが更新されますか。
はい。 お使いの環境該当しない推奨設定は、無効にすることをお勧めします。 特定の推奨事項を無効にする方法については、「[セキュリティ ポリシーの無効化](https://docs.microsoft.com/azure/security-center/tutorial-security-policy#disable-security-policies)」を参照してください。

### <a name="if-a-security-control-offers-me-zero-points-towards-my-secure-score-should-i-ignore-it"></a>特定のセキュリティ コントロールによるセキュア スコアのポイントが 0 の場合、無視すべきですか。
場合によっては、コントロールの最大スコアが 0 を超えているにも関わらず、影響がないことがあります。 リソース修正によるスコアの増分がごくわずかである場合は、0 に丸められます。 これらの推薦事項は修正するとセキュリティが強化されるため、これらの推奨事項は無視しないでください。 唯一の例外は "追加のベストプラクティス" コントロールです。 これらの推奨事項を修復してもスコアは増加しませんが、全体的なセキュリティが強化されます。
