# <a name="azure-managed-applications-overview"></a>Azure マネージ アプリケーションの概要

Azure を使用しているベンダーは、世界中の顧客にソリューションを提供することができます。 Azure Marketplace は、ファーストパーティ ベンダーとサードパーティ ベンダーから提供されている、マルチリソースの複合的なテンプレートが何百と集まって構成されているギャラリーです。 顧客は、サービスとしてのプラットフォーム (PaaS) およびサービスとしてのソフトウェア (SaaS) アプリケーションを数分でデプロイして、使い始めることができます。 

Marketplace は、提供されているソリューションを顧客が短時間でデプロイできるすばらしい場ではありますが、ソリューションの保守と更新は顧客自身が行う必要があります。 アプリケーションの使用については、仮想マシン イメージの使用料以外を、ベンダーが顧客に課金することはできません。 また、ベンダーが、顧客による重要なアプリケーション リソースの改変を防止したり、 アプリケーションを構成している知的財産権へのアクセスを禁止したりすることもできません。 Azure マネージ アプリケーションは、こうした問題の解決策となるものです。 

マネージ アプリケーションは、Marketplace のソリューション テンプレートと似ていますが、1 つ大きな違いがあります。 マネージ アプリケーションでは、ベンダーによって管理されているリソース グループが、リソースのプロビジョニング先となります。 リソース グループは顧客のサブスクリプションに存在しますが、そのリソース グループにアクセスできるのは、ベンダーのテナントの ID です。

## <a name="advantages-of-managed-applications"></a>マネージ アプリケーションの利点

マネージ サービス プロバイダー (MSP)、ISV、および企業の要となる IT チームが、マネージ アプリケーションを使用して、Marketplace またはサービス カタログを通してソリューションを配信できます。 ユーザーは、マネージ アプリケーションを各自のサブスクリプションでデプロイしますが、アプリケーションの管理、更新、またはサービスの提供を行う必要はありません。 ベンダーがアプリケーションを管理およびサポートするため、顧客は、これらのアプリケーションを管理するためのアプリケーション固有のドメインの知識を深める必要はありません。 顧客は、アプリケーションのトラブルシューティングや問題の診断を気にすることなく、アプリケーションの更新プログラムを自動的に取得できます。

ベンダーとプロバイダーについては、マネージ アプリケーションによって、Marketplace を通してインフラストラクチャとソフトウェアの販売が作成されます。 また、マネージ アプリケーションは、サービスと運用サポートを Azure の顧客に付加する方法も提供します。 ベンダーは Azure の課金システムを使って顧客に課金したり、 テンプレートを使用して、デプロイされたアプリケーションのライフサイクルを管理したりできます。 これらのソリューションは自己完結型であり、ユーザーには封印されているため、ベンダーは高品質のサービスを提供できます。 こうしたアプローチは、PaaS ベンダーや SaaS ベンダーに役に立ちます。 また、ソリューションをパッケージ化して再販したいと考えている、企業の要となるプラットフォーム チームやソリューションをシステム インテグレーター (SI) にとっても便利です。

## <a name="managed-application-types"></a>マネージ アプリケーションの種類
Azure のマネージ アプリケーションには、サービス カタログ向けと Marketplace 向けの 2 種類があります。
 
### <a name="service-catalog"></a>サービス カタログ  

サービス カタログを使用すると、顧客が、その組織内で使用される Azure の承認済みソリューションのカタログを作成できます。 そのようなソリューションのカタログを保持することは、企業の要である IT チームにとって有益です。 IT チームはカタログを使用して、組織の特定の基準を守りながら、組織にソリューションを提供します。 ＩＴ チームは、これらのアプリケーションを制御、更新、および管理できます。 従業員はカタログを使用して、IT 部門が推奨し、承認している数多くのアプリケーションを簡単に見つけることができます。 顧客には、自分で作成したサービス カタログ マネージ アプリケーションが表示されます。 組織の他のユーザーと共有しているマネージ アプリケーションを確認することもできます。
 
サービス カタログ マネージ アプリケーションの発行については、「[サービス カタログ マネージ アプリケーションの作成と発行](managed-application-publishing.md)」を参照してください。
 
サービス カタログ マネージ アプリケーションの使用については、「[サービス カタログ マネージ アプリケーションの使用](managed-application-consumption.md)」を参照してください。
 
### <a name="marketplace"></a>Marketplace

マネージ アプリケーションは、Azure Portal で Marketplace を通して入手できます。 ベンダーによって発行されたこうしたアプリケーションは、組織内または組織外の全員が使用できます。 このアプローチにより、MSP、ISV、および SI が各自のソリューションをすべての Azure ユーザーに提供できます。 ユーザーへのメリットは、ソリューションの把握や管理にコストを費やすことなく、複雑なソリューションを活用できる点です。 

現時点では、発行元は、ソリューションをマネージ アプリケーションまたは管理されていないソリューション テンプレートとして提供することができます。 発行されるマネージ アプリケーションの主要コンポーネントには、テンプレート ファイルと UI 定義ファイルがあります。 テンプレート ファイルは、プロビジョニングされるリソースを記述し、 UI 定義ファイルは、これらのリソースをプロビジョニングするために必要な入力がポータルにどのように表示されるかを記述します。 必要なファイルは .zip ファイルにパッケージ化され、発行元のポータルを通してアップロードされます。
 
マネージ アプリケーションを Marketplace に発行する方法については、「[Marketplace の Azure マネージ アプリケーション](managed-application-author-marketplace.md)」を参照してください。

Marketplace からマネージ アプリケーションを使用する方法については、「[Consume Azure managed applications in the Marketplace (Marketplace での Azure マネージ アプリケーションの使用)](managed-application-consume-marketplace.md)」を参照してください。

## <a name="key-concepts"></a>主要な概念

### <a name="managed-resource-group"></a>マネージ リソース グループ
マネージ リソース グループには、テンプレートでプロビジョニングされているすべての Azure リソースが作成されます。 たとえば、ストレージ アカウントの作成にアプライアンスが使用される場合、このリソース グループには、ストレージ アカウント リソースが含まれます。 アプライアンス リソースは、ここには含まれません。

### <a name="appliance-package"></a>アプライアンス パッケージ
発行元は、各種テンプレート ファイルと createUIDefinition ファイルを含んだパッケージを作成します。 具体的には、このパッケージには次のファイルが含まれます。

- **applianceMainTemplate.json**: このテンプレート ファイルでは、アプライアンスによってプロビジョニングされるすべてのリソースが定義されます。 このファイルは、リソースの作成に使用される通常のテンプレート ファイルです。

- **MainTemplate.json**: このテンプレート ファイルでは、アプライアンス リソース (Microsoft.Solutions/appliances) が定義されます。 このリソースに定義されている 1 つの重要なプロパティとして ManagedResourceGroupId があります。 このプロパティは、applianceMainTemplate.json に定義されている実際のリソースが、どのリソース グループを使ってホストされるかを示します。

- **applianceCreateUIDefinition.json**: このファイルは、テンプレートに定義されているパラメーターに必要な UI のレンダリング方法を記述します。

### <a name="authorization"></a>承認
ベンダーが顧客の代理でリソースを管理するために必要なアクセス許可は、発行元が指定する必要があります。 マネージ リソース グループには、このアクセス許可が適用されます。 次の値を設定します。

- **PrincipalID**: マネージ リソース グループへのアクセスを許可する際に使う、ユーザー、グループ、またはアプリケーションの Azure Active Directory (Azure AD) 識別子。 この識別子は、発行元のテナントに属します。

- **RoleDefinitionID**: 前述のプリンシパル ID に割り当てられるロールの Azure AD 識別子。 発行元のテナントに存在するロールベースのアクセス制御の組み込みロールをどれでも使用できます。 詳細については、「[Azure ロールベースのアクセス制御の組み込みロール](../active-directory/role-based-access-built-in-roles.md)」を参照してください。
