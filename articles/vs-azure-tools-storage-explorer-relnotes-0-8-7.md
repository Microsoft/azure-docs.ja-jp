# <a name="microsoft-azure-storage-explorer-087-preview"></a>Microsoft Azure ストレージ エクスプローラー 0.8.7 (プレビュー)
## <a name="overview"></a>概要
この記事には、Azure ストレージ エクスプローラー 0.8.7 プレビュー リリースのリリース ノートが含まれます。

[Microsoft Azure ストレージ エクスプローラー (プレビュー)](./vs-azure-tools-storage-manage-with-storage-explorer.md) は、Windows、macOS、Linux で Azure Storage データを容易に操作できるスタンドアロン アプリです。

## <a name="azure-storage-explorer-087-preview"></a>Azure ストレージ エクスプローラー 0.8.7 (プレビュー)
### <a name="download-azure-storage-explorer-087-preview"></a>Azure ストレージ エクスプローラー 0.8.7 (プレビュー) をダウンロードする
- [Azure ストレージ エクスプローラー 0.8.7 プレビュー (Windows 用)](https://go.microsoft.com/fwlink/?LinkId=708343)
- [Azure ストレージ エクスプローラー 0.8.7 プレビュー (Mac 用)](https://go.microsoft.com/fwlink/?LinkId=708342)
- [Azure ストレージ エクスプローラー 0.8.7 プレビュー (Linux 用)](https://go.microsoft.com/fwlink/?LinkId=722418)

### <a name="new-updates"></a>新しい更新プログラム
* **[アクティビティ]** ウィンドウで更新、ダウンロード、またはコピー セッションの開始時に競合を解決する方法を選ぶことができます。
* タブをポイントすると、ストレージ リソースの完全なパスが表示されます。
* タブをクリックすると、左側のナビゲーション ウィンドウ内の場所と同期されます。

### <a name="fixes"></a>修正
* 修正: ストレージ エクスプローラーは、macOS の信頼できるアプリになりました。
* 修正: Ubuntu 14.04 が再びサポートされるようになりました。
* 修正: サブスクリプションを読み込むときに [アカウントの追加] UI が点滅することがありました。
* 修正: 左側のナビゲーション ウィンドウにすべてのストレージ リソースが一覧表示されないことがありました。
* 修正: [操作] ペインに空の操作が表示されることがありました。
* 修正: 前回終了したセッションからウィンドウ サイズが維持されるようになりました。
* 修正: コンテキスト メニューを使って同じリソースに対して複数のタブを開くことができるようになりました。

### <a name="known-issues"></a>既知の問題
* クイック アクセスは、サブスクリプション ベースのアイテムでのみ動作します。 このリリースでは、ローカル リソースと、キーまたは SAS トークンによってアタッチされたリソースは、サポートされていません。
* リソースの数によっては、クイック アクセスで対象リソースに移動するまでに数秒かかる場合があります。
* BLOB またはファイルのグループを&4; つ以上同時にアップロードすると、エラーが発生する場合があります。
* 検索によって処理されたノードが約 50,000 を超えると、パフォーマンスが低下するか、またはハンドルされない例外が発生する可能性があります。
* macOS でストレージ エクスプローラーを初めて使うとき、キーチェーンにアクセスするためのユーザーのアクセス許可を確認する複数のプロンプトが表示される可能性があります。 プロンプトが繰り返し表示されないように、**[常に許可]** を選ぶことをお勧めします。

## <a name="previous-releases"></a>以前のリリース
### <a name="features"></a>Features (機能)
#### <a name="main-features"></a>主要な機能
* macOS、Linux、Windows バージョン
* サインインして、サブスクリプション別にグループ化されたストレージ アカウントを表示します
    * 組織アカウント、Microsoft アカウント、2FA などを使います
    * プロキシの設定を構成して管理します
    * サインアウトすることでアカウントを削除します
* 次のものを使ってストレージ アカウントに接続します
    * アカウント名とキー
    * カスタム エンドポイント (Azure China を含む)
    * ストレージ アカウントの SAS URI
* Azure Resource Manager およびクラシック ストレージのサポート
* BLOB、BLOB コンテナー、キュー、テーブル、またはファイル共有の SAS キーを生成します
* Shared Access Signatures (SAS) キーで BLOB コンテナー、キュー、テーブル、またはファイル共有に接続します
* BLOB コンテナー、キュー、テーブル、またはファイル共有の保存されているアクセス ポリシーを管理します
* ストレージ エミュレーターを含むローカル開発ストレージ (Windows のみ)
* BLOB コンテナー、キュー、またはテーブルを作成および削除します
* $logs BLOB コンテナーおよび $metrics テーブルを表示します
* 特定の BLOB、キュー、テーブル、またはファイル共有を検索します
* リソースの共有と簡単なアクセスのため、ストレージ アカウントまたはコンテナー、キュー、テーブル、ファイル共有に直接リンクします
* BLOB コンテナー、テーブル、ファイル共有の名前を変更します
* CORS ルールを管理および構成する機能
* ストレージ アカウントのプライマリ キーとセカンダリ キーを簡単にコピーします
* アップロードとダウンロードでのデータの整合性と一貫性の MD5 チェック
* 検索ボックスから BLOB コンテナー、テーブル、キュー、ファイル共有、またはストレージ アカウントを検索します
* 最もよく使うサービスを簡単にアクセスできるようにクイック アクセスにピン留めできます
* 複数のエディターを別のタブで開けるようになりました。 シングルクリックで一時的なタブが開き、ダブルクリックで永続的なタブが開きます。 一時的なタブをクリックして永続的なタブにすることもできます
* アップロードとダウンロードでのパフォーマンスと安定性の大幅な向上 (特に高速コンピューターでの大きいファイルの場合)
* 拡張された範囲指定検索の再導入と、範囲指定の概念の追加。 ホバー アイコンで右クリックして [ここから検索] を選んでノードのパスを入力するか、手動で入力して、そのノードを範囲指定します。 範囲指定した後は、パスの末尾に検索用語を追加してそのノードから詳細な検索を実行できます。
* さまざまなテーマを追加しました。明るい (既定値)、暗い、ハイコントラスト黒、ハイ コントラスト白。
* [編集] の [テーマ] を選んでテーマの設定を変更します。
* Linux では、64 ビット OS が必要になりました
* ロゴを更新しました。
#### <a name="blobs"></a>BLOB
* BLOB を表示し、ディレクトリ内を移動します
* BLOB とフォルダーをアップロード、ダウンロード、削除、コピーします
* コンテンツ テキストと画像 BLOB を開いて表示します
* BLOB のプロパティとメタデータを表示および編集します
* プレフィックスで BLOB を検索します
* BLOB および BLOB コンテナーのリースを作成および解除します
* ドラッグ アンド ドロップでファイルをアップロードします
* BLOB とフォルダーの名前を変更します
* 空の "仮想" フォルダーを BLOB コンテナーに作成できます
* BLOB とファイルのプロパティを変更できます
#### <a name="tables"></a>テーブル
* ODATA でエンティティを表示またはクエリしたり、クエリ ビルダーを使って複雑なクエリを作成します
* エンティティを追加、編集、削除します
* CSV 形式でテーブルの内容をインポートおよびエクスポートします (クエリ結果のエクスポートを含む)
* 列の順序をカスタマイズします
* クエリを保存する機能
#### <a name="queues"></a>キュー
* 最新の 32 個のメッセージをピークします
* メッセージを追加、デキュー、表示します
* キューをクリアします
* キュー メッセージをエンコード/デコードするかどうかを決定できます
#### <a name="file-shares"></a>ファイル共有
* ファイルを表示し、ディレクトリ内を移動します
* ファイルとディレクトリをアップロード、ダウンロード、削除、コピーします
* ファイルのプロパティを表示します
* ファイルとディレクトリの名前を変更します

### <a name="bug-fixes"></a>バグの修正
* 修正: 画面がフリーズする問題
* 修正: セキュリティの強化

### <a name="known-issues"></a>既知の問題
* 検索によって処理されたノードが約 50,000 を超えると、パフォーマンスが低下します。macOS のインストールに管理者特権が必要な場合があります
* サブスクリプションをフィルターするために資格情報の再入力が必要であることがアカウント設定パネルに表示されることがあります
* BLOB の名前の変更で (個別または名前を変更する BLOB コンテナーの内部)、スナップショットが保持されません。 BLOB、ファイル、エンティティの他のすべてのプロパティとメタデータは、名前変更の間に保持されます。
* Azure Stack は現在ファイルをサポートしないので、**[ファイル]** ノードを展開しようとするとエラーになります
* Linux 14.04 のインストールには、更新またはアップグレードされた gcc のバージョンが必要です。 次の手順では、アップグレードする方法を示します。

```
sudo add-apt-repository ppa:ubuntu-toolchain-r/test
sudo apt-get update
sudo apt-get upgrade
sudo apt-get dist-upgrade
```
