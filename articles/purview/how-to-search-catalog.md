# <a name="search-the-azure-purview-data-catalog"></a>Azure Purview Data Catalog を検索する

この記事では、Azure Purview Data Catalog のさまざまな検索機能の使用方法について説明します。

## <a name="search-the-catalog-for-assets"></a>カタログで資産を検索する

資産の検索を実行する手順は、次のとおりです。

1. **[カタログの検索]** を選択して、[資産検索ダイアログを開きます](#open-the-asset-search-dialog)。
1. [検索語句を入力](#enter-search-terms)して、その語句と一致する特性を持つ資産を検索します。
1. 検索を絞り込むには、[クイック フィルターを設定](#set-quick-filters)します。
1. [検索を開始](#start-the-search)し、検索結果にアクセスします。

クイック フィルターは、検索語句の入力の前または後のどちらで設定しても問題ありません。

検索語句を入力せず、フィルターも設定しない場合、検索結果にはすべての資産が含まれます。

### <a name="open-the-asset-search-dialog"></a>資産検索ダイアログを開く

**[カタログの検索]** を選択して、資産検索ダイアログを開きます。

:::image type="content" source="./media/how-to-search-catalog/search-catalog.png" alt-text="[カタログの検索] の下の左側には、検索フィルターを表示したペインがあり、右側には、最近の検索を表示したペインがあります。" border="true":::

検索ダイアログには、クイック フィルター、検索履歴、最近アクセスした資産の一覧が表示されます。

:::image type="content" source="./media/how-to-search-catalog/asset-search-dialog.png" alt-text="[カタログの検索] の下にある右側のペインには、検索の一覧が表示されます。" border="true":::

### <a name="enter-search-terms"></a>検索語句を入力する

**[カタログの検索]** に 1 つ以上の検索語句を入力します。 入力すると、 **[最近実行した検索]** に、最近の検索と一致する検索語句の一覧が表示され、 **[検索候補]** に、候補の一致する検索語句の一覧が表示され、さらに、 **[資産候補]** に、一致するデータ資産の一覧が表示されます。

:::image type="content" source="./media/how-to-search-catalog/enter-search-terms.png" alt-text="[カタログの検索] に入力された検索の結果を示すスクリーンショット":::

検索結果には、検索語句と一致する 1 つ以上の特性を持つ資産のみが含まれます。 これらの特性としては、資産名、資産の種類、分類、連絡先があります。

#### <a name="types-of-search-criteria"></a>検索条件の種類

Azure Purview では、次の種類の検索条件がサポートされます。

> [!Note]
> ブール演算子 (**AND**、**OR**、**NOT**) は、常にすべて大文字で指定します。 それ以外については、大文字と小文字の区別はなく、余分なスペースも問題にはなりません。

- **hive**: **hive** を含むドキュメントを検索します。
- **hive データベース**: **hive データベース** のみを含むドキュメントを検索します。
- **hive OR データベース**: **hive** または **データベース**、またはその両方を含むドキュメントを検索します。
- **hive AND データベース**、**hive && データベース**: **hive** と **データベース** の両方を含むドキュメントを検索します。
- **hive AND (データベース OR ウェアハウス)** : **hive** と、**データベース** または **ウェアハウス** のいずれか、またはその両方を含むドキュメントを検索します。
- **hive NOT データベース**: **hive** を含むが、**データベース** を含まないドキュメントを検索します。
- **hiv**: **hiv** で始まる語を含むドキュメントを検索します。 たとえば、**hiv**、**hive**、**hivbar** などです (* は、任意の数の文字と一致するワイルド カードです)。

### <a name="set-quick-filters"></a>クイック フィルターを設定する

検索結果リストは、 **[カタログの検索]** に入力した検索語句、およびクイック フィルターで選択した値に基づきます。

クイック フィルターにより、検索結果リストは、特定の選択された値を含む資産に制限されます。 フィルターには、ドロップダウン リストとテキスト ボックスがあります。 ドロップダウン リストには、"*現在*" の検索結果に含まれる特性の値が表示されます。 リスト内の各値の横には、現在の検索結果内にある、その値を持つ資産の数のカウントが表示されます。 リストから値を選択すると、検索結果は、その値を持つ資産に制限されます。 選択できる値は 1 つだけです。

ドロップダウン リストの作成に使用される現在の検索結果は、次のもので決定されます。

- **[カタログの検索]** に入力された検索語句。 
- クイック フィルターで選択された値。

[資産の種類] クイック フィルターの例を次に示します。

:::image type="content" source="./media/how-to-search-catalog/asset-type-quick-filter.png" alt-text="[資産の種類] クイック フィルターの例。" border="true":::

テキスト ボックスにテキストを入力して、ドロップダウン リスト内の値を、テキストまたはその一部と一致する値に制限することができます。 テキスト ボックスの使用例については、「[検索クイック フィルター: 資産の種類によるフィルター処理](#search-quick-filter-filter-by-asset-type)」、および「[検索クイック フィルター: 分類によるフィルター処理](#search-quick-filter-filter-by-classification)」を参照してください。

#### <a name="search-quick-filter-filter-by-asset-type"></a>検索クイック フィルター: 資産の種類によるフィルター処理

資産の種類でフィルター処理するには、 **[資産の種類]** クイック フィルターを使用します。 ドロップダウン リストには、検索語句とクイック フィルターで決定された現在の検索結果内で見つかった資産の種類が表示されます。 種類ごとに、その種類の資産の数が表示されます。

:::image type="content" source="./media/how-to-search-catalog/asset-type-quick-filter.png" alt-text="[資産の種類] クイック フィルターが強調表示されています。ここには、資産の種類とそれぞれのカウントが表示されます。" border="true":::

検索結果をその種類の資産に制限するには、資産の種類を選択します。 選択できる種類は 1 つだけです。

文字列と一致する名前を持つ資産の種類のみを表示するには、テキスト ボックスに文字列を入力します。 たとえば、名前に **sql** を含む資産の種類のみを表示するには、「**sql**」と入力します。

:::image type="content" source="./media/how-to-search-catalog/filter-asset-types.png" alt-text="[クイック フィルター] ペインの [資産の種類] には、sql が入力されています。sql を含む資産の一覧には、3 つのエントリが表示されています。" border="true":::

検索結果をその種類の資産に制限するには、資産の種類を選択します。 選択できる種類は 1 つだけです。

#### <a name="search-quick-filter-filter-by-classification"></a>検索クイック フィルター: 分類によるフィルター処理

資産の分類でフィルター処理するには、 **[分類]** クイック フィルターを使用します。 ドロップダウン リストには、検索語句とクイック フィルターで決定された現在の検索結果内にある、1 つ以上の資産に割り当てられた分類が表示されます。 分類ごとに、その分類を割り当てられた資産の数が表示されます。

:::image type="content" source="./media/how-to-search-catalog/classification-quick-filter.png" alt-text="[分類] クイック フィルターが強調表示されています。" border="true":::

分類を選択して、検索結果を、その分類を割り当てられた資産に制限します。 選択できる分類は 1 つだけです。

文字列と一致する名前を持つ分類のみを表示するには、テキスト ボックスに文字列を入力します。 たとえば、名前に **番号** を含む分類のみを表示するには、「**番号**」と入力します。

:::image type="content" source="./media/how-to-search-catalog/filter-classifications.png" alt-text="[クイック フィルター] ペインでは、[分類] は '番号' であり、その値を含むすべての分類が一覧表示されています。" border="true":::

分類を選択して、検索結果を、その分類を割り当てられた資産に制限します。 複数の分類を選択することはできません。

#### <a name="search-quick-filter-filter-by-contacts"></a>検索クイック フィルター: 連絡先によるフィルター処理

"*連絡先*" は、所有者またはエキスパートとして資産に割り当てられた人物です。 資産の詳細を表示すると、連絡先が **[連絡先]** タブに表示されます。

特定の連絡先が割り当てられた資産を検索するには、2 つの方法があります。

- 連絡先の全部または一部を **[カタログの検索]** に入力し、検索を実行します。 検索結果には、連絡先の名前が検索語句と一致する資産が含まれます。
- 目的の連絡先を **[連絡先]** クイック フィルターで選択し、検索を実行します。

:::image type="content" source="./media/how-to-search-catalog/contact-quick-filter.png" alt-text="[クイック フィルター] ペインの [連絡先] の値は、'Vishal' です。[候補] ペインには、3 つの候補があります。" border="true":::

## <a name="search-example"></a>検索例

検索用語とクイック フィルターがどのようにやりとりして検索結果を決定するのかを確認するために、架空の例を見てみることにしましょう。 特に、資産の種類 **[Azure Blob Storage]** のカウントを監視します。

- 検索語句を入力せず、クイック フィルターでも値を選択しないで、1 回目の検索を実行します。 この検索では、カタログ内のすべての資産が検出されます。 検索結果リストと **[資産の種類]** クイック フィルターには、次の内容が表示されます。

    - 検索結果リストには、164,230 個の資産 (カタログ内のすべての資産) が含まれます。
    - **[資産の種類]** ドロップダウン リストには、43 個のエントリが含まれます。 これらは、カタログ内のすべての資産の種類です。 1 つのアセットは 1 つの種類のみであるため、43 個のアセットの種類の合計カウントは 164,230 になります。
    - 資産の種類 **[Azure Blob Storage]** は、 **[資産の種類]** クイック フィルターのドロップダウン リストで最初のエントリです。 値は、最も大きいカウントからカウント順に順序付けされるため、**Azure Blob Storage** は最も一般的な資産の種類です。 そのカウントは、118,174 です。

- 次に、 **[カタログの検索]** に 「**parquet**」と入力し、別の検索を実行します。 検索結果には、**parquet** と一致する特性を持つ資産のみが含まれます。 これにより、すべてのカウントは次のようにして削減されます。

    - 検索結果リストには、493 個の資産が含まれます。 カタログ内の 164,230 個の資産のうち 493 個のみが、"parquet" と一致する特性を持ちます。
    - **[資産の種類]** ドロップダウン リストには、15 個のエントリが含まれます。 493 個のアセットはすべて、これらの 15 種類のいずれか 1 つであり、15 種類の合計カウントは 493 です。
    - 種類が **[Azure Blob Storage]** の資産は 456 個あります。 残りの 37 (493 - 456) 個の資産は、残りの 14 種類のいずれかです。

- ここで、別のクイック フィルター **[分類]** のドロップダウン リストを見てみましょう。

    - 検索結果リストには、493 個の資産に対して 12 の分類があります。 資産には任意の数の分類を割り当てることができるため、493 個の資産の合計カウントは 493 にはなりません。
    - 分類 **[人物の名前]** は、他の分類よりも多い 36 個の資産に割り当てられています。

- 分類 **[人物の名前]** を選択します。 **[人物の名前]** のカウントは 36 であったため、予想どおり、検索結果リストの資産数は 36 個に減少します。 これらの結果のいずれも、種類は **[Azure Blob Storage]** ではありません。

種類が **[Azure Blob Storage]** で、**parquet** に一致し、分類が **[人物の名前]** である資産はないと結論付けることができます。

## <a name="start-the-search"></a>検索を開始する

検索すると、 **[カタログの検索]** に入力した検索語句が資産の特性と照合されます。 これらの特性としては、資産の種類、分類、連絡先があります。 特性が一致する資産は、クイック フィルターで除外されない限り、検索結果リストに表示されます。

検索語句を入力してクイック フィルターを設定した後、次のいずれかの方法で検索を開始します。

- 入力した用語に基づいて検索するには、検索アイコン (:::image type="icon" source="./media/how-to-search-catalog/search-icon.png":::) を選択して **Enter** キーを押すか、 **[検索結果の表示]** を選択します。
- 以前の検索の語句を使用して検索するには、 **[最近実行した検索]** から選択します。
- 候補の語句を使用して検索するには、 **[検索候補]** から選択します。

**[検索候補]** から資産を選択すると、資産の詳細ページに直接移動します。 検索は実行されません。

候補とユーザー検索の結果リストは、多少異なる場合があります。 候補リストの結果は、あいまい一致に基づき、ユーザーが開始した検索の結果は、完全一致に基づきます。

検索すると、 **[検索結果]** ページが表示され、検索で見つかった資産の一覧が表示されます。

:::image type="content" source="./media/how-to-search-catalog/search-results.png" alt-text="検索値 contoso の検索結果を示すスクリーンショット。":::

資産の詳細を表示するには、資産名を選択します。

他の検索結果ページに移動するには、検索結果ページの下部にあるコントロールを使用します。

:::image type="content" source="./media/how-to-search-catalog/page-navigation.png" alt-text="検索結果ページを移動する方法を示すスクリーンショット。":::

### <a name="sort-search-results"></a>検索結果を並べ替える

**[関連性]** または **[名前]** で検索結果を並べ替えるには、 **[並べ替え]** を使用します。

:::image type="content" source="./media/how-to-search-catalog/sort-by.png" alt-text="検索結果を並べ替える方法を示すスクリーンショット。この例の場合、[並べ替え] ドロップダウン リストは [関連性] に設定されています。":::

### <a name="search-results-dynamic-filters"></a>検索結果の動的フィルター

**[検索結果]** ページの **[フィルター]** ペインには、検索結果リストの資産を動的にフィルター処理するフィルターがあります。 フィルター処理は動的であり、フィルターの選択に基づいて追加のフィルターを表示できます。

動的フィルターには、ドロップダウン リストの値ごとにチェック ボックスがあります。 これらのチェック ボックスを使用して、必要な数の値をフィルター処理します。

#### <a name="search-results-dynamic-filter-filter-by-asset-type"></a>検索結果の動的フィルター: 資産の種類によるフィルター処理

**[資産の種類]** ドロップダウン リストで資産の種類を選択すると、動的フィルターが表示され、検索結果を絞り込むための追加の方法が提供されます。 フィルターは、選択した資産の種類によって異なります。 たとえば、 **[Azure SQL Database]** を選択すると、サーバー、データベース、スキーマに対して動的フィルターが表示されます。 これらのフィルターの値は、選択した種類の検索結果内にある資産の値です。

:::image type="content" source="./media/how-to-search-catalog/asset-type-dynamic-filter.png" alt-text="Azure SQL Database フィルター項目は、選択した [資産の種類] 項目のみです。その資産の種類の検索結果も強調表示されています。" border="true":::

#### <a name="search-results-dynamic-filter-filter-by-classification"></a>検索結果の動的フィルター: 分類によるフィルター処理

**[分類]** リストの各分類は、検索結果リストの少なくとも 1 つの項目に適用されます。 検索結果を、選択した分類の資産に絞り込むには、1 つ以上の分類を選択します。

:::image type="content" source="./media/how-to-search-catalog/classification-dynamic-filter.png" alt-text="[検索結果] の [分類] フィルターが強調表示されています。" border="true":::

#### <a name="edit-and-delete-search-results-filters"></a>検索結果フィルターの編集と削除

フィルターを削除するには、フィルター名の横にあるチェック ボックスをクリアします。

### <a name="recently-accessed-assets"></a>最近アクセスした資産

展開された検索ボックスの **[最近アクセスした]** セクションには、最近アクセスした資産 (ある場合) が表示されます。

- 最近アクセスした資産の完全なリストを表示するには、 **[最近アクセスした]** セクションで **[すべて表示]** を選択します。

   :::image type="content" source="./media/how-to-search-catalog/get-to-recent-view.png" alt-text="展開した検索ボックスの [最近アクセスした] セクションを示すスクリーンショット。":::

   最近アクセスした資産の一覧が表示されます。

   :::image type="content" source="./media/how-to-search-catalog/recent.png" alt-text="最近アクセスした資産の一覧を示すスクリーンショット。":::

- 名前でフィルター処理するには、 **[名前でフィルター処理]** に検索文字列を入力します。

- 一覧から項目を削除するには、それらのチェック ボックスをオンにして、 **[削除]** を選択します。

   :::image type="content" source="./media/how-to-search-catalog/remove-from-recent-view.png" alt-text="最近アクセスした資産の一覧から項目を削除する方法を示すスクリーンショット。":::

- 一覧全体をクリアするには、 **[クリア]** を選択します。

   :::image type="content" source="./media/how-to-search-catalog/clear-recent-view-selections.png" alt-text="最近アクセスした資産の一覧をクリアする方法を示すスクリーンショット":::

### <a name="search-assets"></a>資産を検索する

**[ホーム]** 以外の多くのページには、上部に **[資産の検索]** ボックスがあります。 たとえば、次の図は資産の詳細ページで、 **[資産の検索]** が強調表示されています。

:::image type="content" source="./media/how-to-search-catalog/search-assets.png" alt-text="[資産の検索] が強調表示された資産の詳細ページを示すスクリーンショット":::
