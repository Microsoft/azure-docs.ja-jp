# <a name="publishing-integrated-solutions"></a>統合ソリューションの公開

統合ソリューションは、複数の ISV ポイント ソリューション テクノロジをシステム インテグレーター (SI) のデプロイとサービスと合わせて 1 つのバンドル オファーに統合したもので、SI によって[コンサルティング サービス オファー](https://docs.microsoft.com/azure/marketplace/consulting-services)としてマーケットプレースに掲載されます。

これらのソリューションは、お客様の特定のニーズまたはシナリオに対応するために構築され、その構築は、業界の課題や問題点を理解することから始まります。 その後、一元管理された再現性のあるデプロイ エクスペリエンスを作成し、お客様の迅速な立ち上げと運用を支援するために、SI によって 2 つ以上の最善のテクノロジーが統合されされます。 システム インテグレーターは、お客様との単一窓口としても機能し、ソリューションに関連するカスタマイズ、コンサルティング、およびサポートを提供します。 その成果が、テクノロジとサービスを含む統合ソリューションです。

[統合ソリューション プログラム](https://aka.ms/AA5qos4)の概要については、こちらのビデオをご覧ください。

## <a name="key-benefits-to-integrated-solutions-in-the-marketplace"></a>マーケットプレースにある統合ソリューションの主なメリット

* お客様は、エンドツーエンドのビジネス上の課題を解決するパッケージ ソリューションをマーケットプレースを通じて見つけることができるほか、事前に統合されたテクノロジにより価値実現までの時間を短縮することができます。 お客様は、断片的にデプロイされた複数の ISV アプリを個別に取り扱わなければならない状態を回避できます。
* ISV は、自社の IP を、そのテクノロジの価値を高める他のソリューションやサービスと共にパッケージ化し、エキスパートの SI を通じてデリバリーします。
* SI は、より多くの顧客アカウントでのスケーリング、ランディング、展開を実現するための最善のテクノロジを使用した、事前に定義および統合されたソリューションを作成します。
* Microsoft は、これらの事前に統合された業界に合わせたソリューションを活用して、価格と条件を事前に分類し、より簡単にお客様に共同販売できます。 

## <a name="eligibility-criteria"></a>対象となる条件

* 参加するパートナーすべてに MPN ID が必要です
* 参加する SI は、共同販売の準備ができている必要があります
* 参加する ISV は、IP 共同販売である必要があります 
* 少なくとも 1 社の参加パートナーが、次のいずれかの分野で Gold コンピテンシーを取得している必要があります。

    * アプリケーション開発
    * アプリケーションの統合
    * アプリケーション ライフサイクル管理
    * クラウド プラットフォーム
    * データ分析
    * データ センター
    * データ プラットフォーム
    * DevOps

* ソリューションに少なくとも 2 つの取引先組織が含まれ、完全な顧客ソリューションとエクスペリエンスのために IP とサービスが統合されている必要があります
* パートナー IP は、Azure、Dynamics 365、Power Platform のいずれかに構築されている必要があります
* 統合ソリューションの IP は、単なる "ISV の IP + 基本 SI サービス" とは区別する必要があります
* すべてのパートナーにわたる目標は、Azure Marketplace または AppSource で掲載されて最終的に取り引きされる、反復可能なオファーを作成することです

## <a name="nomination-process"></a>申請プロセス

このプログラムへの参加を希望するパートナーは、連携したい他のパートナーを特定したうえで、[統合ソリューションの申請](https://aka.ms/AA5qicu)フォームに記入する必要があります。 その後、統合ソリューション プログラム チームが申請を確認し、10 営業日以内に承認と次の手順に関するフォローアップを行います。 

## <a name="developing-your-integrated-solution"></a>統合ソリューションの開発 

申請が承認されたら、リンク先のプログラム リソースを活用しながら、以下の手順に従って統合ソリューションを開発します。 

1. ワークショップ形式で主要なビジネスおよび技術に関するディスカッションを行い、ソリューション (スコープ、対象顧客、価値提案、価格設定、ライセンス、利用規約、サポート方法、マーケティング活動、営業プロセス、およびアクティビティ) と統合方法 (統合ポイント、ソリューション アーキテクチャ、認証、ユーザー インターフェイス、監視、デプロイ メカニズム、API、および統合データ リポジトリ) を定義します。 

    パートナーにこのプロセスをガイドするために、次のリソースが用意されています。

    1. [ビジネスに関する意思決定ワークショップのディスカッション ガイド](https://aka.ms/AA5qicx)
    1. [技術に関する意思決定ワークショップのディスカッション ガイド](https://aka.ms/AA5qid1)
    1. [クイックスタート ビデオ - ビジネスおよび技術に関するワークショップ](https://aka.ms/AA5qos9)

1. ビジネスおよび技術に関するワークショップの結果としてのプロジェクト計画とタイムラインを推進するために、クロスパートナーのプロジェクト マネージャーを特定します。

1. ソリューションの技術的な統合を完了します。

1. ソリューションの価格を確定し、Azure Marketplace または AppSource に掲載する単一の価格ポイントを決定します。

1. Azure Marketplace または AppSource への掲載用に、以下を含むマーケティング資料を完成させます。

    1. 統合ソリューションの名前
    2. 統合ソリューションの掲載用の説明 ([オファーの掲載のベスト プラクティス](https://docs.microsoft.com/azure/marketplace/gtm-offer-listing-best-practices))
    1. 顧客 1 のページャー ([テンプレート](https://aka.ms/AA5s08a))
    1. 顧客プレゼンテーション ([テンプレート](https://aka.ms/AA5s7ql))
    1. 統合ソリューションのロゴ 

## <a name="publishing-your-integrated-solution"></a>統合ソリューションの公開 

技術的な統合が完了し、マーケティング資料の準備ができたら、[Azure Marketplace および AppSource 向けコンサルティング サービス](https://docs.microsoft.com/azure/marketplace/consulting-services)の公開元ガイドを参照してください。 このリソースを使用して、リーチしたい対象ユーザーに応じて Azure Marketplace と AppSource のどちらで公開するかを決定し、公開する成果物を準備して、公開プロセスを完了します。

コンサルティング サービス オファーには 5 種類のサービスが用意されていますが、統合ソリューションは概念実証または完全な実装のいずれかである必要があります。

## <a name="going-to-market-with-your-integrated-solution"></a>統合ソリューションで市場に参入する 

ソリューションが Azure Marketplace または AppSource に表示されるようになったら、マーケティング活動を通じて認知度と需要の創出を促進することができます。 [プロジェクト計画とタイムライン](https://aka.ms/AA5qiuc)に関するページに記載されている活動を確認します。これには、パートナーが自分のチーム内で促進できる活動と、互いに促進できる活動が含まれます。 次に、マーケットプレースでの公開を行ったパートナーに提供される [Go-to-Market ベネフィット](https://docs.microsoft.com/azure/marketplace/gtm-your-marketplace-benefits#list-trial-and-consulting-benefits)にアクセスします。 すべての統合ソリューション オファーは、試用版レベルのベネフィットの対象となります。
