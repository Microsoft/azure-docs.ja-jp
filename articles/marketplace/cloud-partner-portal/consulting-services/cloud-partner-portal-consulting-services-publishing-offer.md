# <a name="azure-and-dynamics-365-consulting-service-offer"></a>Azure と Dynamics 365 のコンサルティング サービス プラン

このセクションでは、<a href="https://azuremarketplace.microsoft.com">Azure Marketplace</a> または <a href="https://appsource.microsoft.com">AppSource Marketplace</a> のいずれかにコンサルティング サービス プランを発行する方法について説明します。 Microsoft <a href="https://dynamics.microsoft.com">Dynamics 365</a>、<a href="https://powerbi.microsoft.com">Power BI</a>、<a href="https://powerapps.microsoft.com">PowerApps</a> に基づくソリューションは、AppSource に掲載される資格があります。 その他の Microsoft <a href="https://azure.microsoft.com/services">Azure サービス</a>に基づくその他のオファーは、Azure Marketplace に掲載される資格があります。

## <a name="publishing-benefits"></a>公開の利点

Microsoft のいずれかのマーケットプレースに公開することには、大きな利点があります。

- Microsoft のブランドを活用することで、会社の宣伝ができる。
- AppSource で 1 億を超える Office 365 と Dynamics 365 のユーザー、Azure Marketplace を通じて 20 万を超える組織にリーチできる可能性がある。
- これらのマーケットプ レースから質の高い潜在顧客を獲得できる。
- Microsoft のフィールド チームとテレセールス チームによりサービスを宣伝してもらえる。

## <a name="publish-a-consulting-service-offer"></a>コンサルティング サービス プランの発行

このセクションでは、コンサルティング サービス プランの発行に伴う各種の要素の概要を説明します。 発行は、次の主要な部分に分かれています。

1. [コンサルティング サービスの前提条件](./cpp-consulting-service-prerequisites.md) - **Dynamics 365 for Customer Engagement**、**Dynamics 365 for Finance & Operations**、**Dynamics 365 Business Central**、**Power BI**、**PowerApps** 向けのコンサルティング サービス プランを発行するための要件を示します。
1. [新しいプランの作成](./cpp-consulting-service-create-offer.md) - Cloud パートナー ポータルを使用してコンサルティング サービス プラン エントリを作成するために必要な手順を示します。
    1. [プランの設定を定義する](./cpp-consulting-service-define-offer-settings.md)
    1. [ネットショップの詳細を入力し、Azure Marketplace と AppSource のどちらで発行するかを入力する。](./cpp-consulting-service-storefront-details.md)
1. [プランを発行する](./cpp-consulting-service-publish-offer.md) - 発行するプランを送信する方法について説明します。

## <a name="next-steps"></a>次の手順

これらの手順でコンサルティング サービス プランを発行する前に、コンサルティング サービス プランを発行するための[前提条件](./cpp-consulting-service-prerequisites.md)を満たす必要があります。
