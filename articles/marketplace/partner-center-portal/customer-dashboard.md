# <a name="customer-dashboard-in-commercial-marketplace-analytics"></a>コマーシャル マーケットプレース分析の [顧客] ダッシュボード

この記事では、パートナー センターの **[顧客] ダッシュボード**について説明します。 このダッシュボードには、成長傾向などの顧客に関する情報が、グラフィカルかつダウンロード可能な形式で表示されます。

**[顧客] ダッシュボード**にアクセスするには、コマーシャル マーケットプレースにある **[[分析]](https://partner.microsoft.com/dashboard/commercial-marketplace/analytics/summary)** ダッシュボードを開きます。

>[!NOTE]
> 分析の用語の詳細な定義については、「[コマーシャル マーケットプレース分析に関するよく寄せられる質問と用語](./faq-terminology.md)」を参照してください。

## <a name="customer-dashboard"></a>顧客ダッシュボード

**[分析]** メニューの **[顧客] ダッシュボード**には、プランを購入した顧客のデータが表示されます。 次の項目のグラフィカル表示を見ることができます。

- [顧客の概要](#customer-summary)
- [地域別の顧客](#customer-by-geography)
- [顧客の傾向](#customer-trends)
- [注文別および使用量別の顧客](#customers-by-orders-and-usage)
- [プラン別の顧客](#customers-by-plans)
- [顧客タイプごとの注文数と使用量](#orders-and-usage-by-customer-type)
- [顧客詳細テーブル](#customer-details-table)
- [顧客ページのフィルター](#customer-page-filters)

顧客の獲得からパートナー センターでの報告までの最大待ち時間は、48 時間です。

### <a name="customer-summary"></a>顧客の概要

[顧客の概要] セクションには、選択した日付範囲内の新規、既存、チャーンを含むすべての顧客の数が表示されます。

- 顧客総数は、プランを購入したことがあり、キャンセルしていない注文が少なくとも 1 つはある顧客の総数として定義されます。
- 前月比の顧客増減率は、数字と緑の上向きインジケーターまたは赤の下向きインジケーターで示されます。
- 増加傾向は棒グラフで表され、グラフの列の上にポインターを合わせると各月の値が表示されます。

**顧客の種類**には、新規、既存、チャーンの 3 つがあります。

- 新規顧客は、選択した月のうちに初めて 1 つ以上オファーを購入した顧客です。
- 既存顧客は、選択した月よりも前に 1 つ以上オファーを購入したことがある顧客です。
- チャーン顧客は、以前に購入したすべてのオファーをキャンセルした顧客です。

### <a name="customer-by-geography"></a>地域別の顧客

**[地域別の顧客]** グラフには、顧客の総数と選択した日付範囲内に獲得した顧客数が表示され、顧客の国や地域に基づいてマップされます。 マップ上の明るい色は顧客数が少ないことを表し、暗い色は顧客数が多いことを表します。 テーブル内のレコードを選択すると、1 つの国や地域が拡大されます。

ヒートマップには、顧客の国や地域ごとの顧客数と割合が表示されます。 マップを移動して正確な場所を表示したり、特定の場所を拡大したりできます。 このマップには補助グリッドが用意されており、その場所に新たに追加された顧客に加えて、場所別に顧客をパーセント表示できます。

### <a name="customer-trends"></a>顧客傾向

**[Customer trends]\(顧客傾向\)** グラフには、新規、既存、チャーンを含むすべての顧客の数が、月ごとの増加傾向と共に表示されます。

- 折れ線グラフは、顧客全体の増加率を表します。
- 月の列は、新規、既存、チャーンの各顧客を積み重ねた顧客数を表します。
- チャーン顧客数は Y 軸の負の方向に表示されます。
- 特定の凡例項目を選択して、より詳細なビューを表示できます。 たとえば、新規顧客のみを表示するには、凡例から新規顧客を選択します。
- グラフの上部にあるスライダーを使用して、X 軸で左右にスクロールし、特定のデータ ポイントにフォーカスして詳細を表示することができます。
- グラフの列にポインターを合わせると、その月の詳細のみが表示されます。

### <a name="customers-by-orders-and-usage"></a>注文別および使用量別の顧客

**[Customers by orders/usage] (注文別および使用量別の顧客)** グラフには、[orders] (注文)、[normalized usage] (正規化された使用量)、[raw usage] (未調整の使用量) の 3 つのタブがあります。 **[top customer percentile]\(上位顧客のパーセンタイル\)** は注文数によって決定され、X 軸に沿って表示されます。 Y 軸には顧客の注文数が表示されます。 第 2 の Y 軸 (折れ線グラフ) には、合計注文数に対する累積の割合が表示されます。 折れ線グラフ上の点にポインターを合わせると、詳細を表示できます。

例として、正規化された使用量のグラフを次に示します。顧客の上位 30 パーセンタイルが、累積では正規化された使用量の 87% に貢献しています。 顧客の 30 パーセンタイルが、使用時間では 157 万時間のみ貢献しています。

### <a name="customers-by-plans"></a>プラン別の顧客

**[Customers by plans/usage]\(プラン別および使用量別の顧客\)** グラフについて、次に説明します。

1. リーダー ボードには、注文数によって順位付けされた上位 50 の顧客の詳細が表示されます。 顧客を選択すると、このリーダー ボードのセクション 2、3、4 にその顧客の詳細が表示されます。
2. 顧客プロファイルの詳細は、発行者が所有者ロールを使用してログインしている場合、この領域に表示されます。 パブリッシャーが共同作成者ロールを使用してログインしている場合、このセクションの詳細は使用できません。
3. **[Orders by plans]\(プラン別の注文数\)** ドーナツ グラフには、プランに対して購入された注文の内訳が表示されます。 注文数が最も多い上位 5 プランが表示され、残りの注文は [rest all] (残りすべて) にまとめられます。
4. **[Seats by plans]\(プラン別のシート\)** ドーナツ グラフには、プランに対して注文されたシートの内訳が表示されます。 シート数が最も多い上位 5 プランが表示され、残りの注文は [rest all] (残りすべて) にまとめられます。

### <a name="orders-and-usage-by-customer-type"></a>顧客タイプごとの注文数と使用量

**[Orders/usage by customer type]\(顧客タイプごとの注文数/使用量\)** グラフには、注文数、正規化された使用時間、および生の使用時間が、新しい顧客と既存の顧客とを分けて、グラフで [orders]\(注文\)、[normalized usage]\(正規化された使用\)、[raw usage]\(未調整の使用\) のいずれを選択したかに応じて表示されます。

- 新規顧客は、同じ月 (Y 軸) のうちに初めて 1 つ以上オファーを購入したか、使用を報告した顧客です。
- 既存顧客は、レポート内の月 (Y 軸) よりも前にプランを購入したか、使用を報告したことがある顧客です。

### <a name="customer-details-table"></a>顧客詳細テーブル

**顧客詳細**テーブルには、上位 1000 顧客の番号付きリストが、プランの 1 つを最初に購入した日付で並べ替えて表示されます。

- 顧客の個人情報は顧客が同意した場合にのみ参照できます。 この情報は、所有者ロール レベルのアクセス許可でログインしている場合にのみ表示できます。 ユーザーのロールとアクセス許可に関する詳細をご確認ください。
- グリッドの各列で並べ替えることができます。
- レコード数が 1000 未満の場合、TSV ファイルにデータを抽出できます。
- レコード数が 1000 を超える場合、エクスポートされたデータは今後 30 日間、ダウンロード ページに非同期で配置されます。
- 関心のあるデータのみを表示するには、テーブルにフィルターを適用します。 データは、会社名、顧客 ID、Marketplace サブスクリプション ID、Azure ライセンスの種類、購入日、失効日、顧客の電子メール、顧客の国、地域、都道府県、市区町村、郵便番号、顧客の言語などでフィルター処理できます。
- 保護された顧客によってオファーが購入された場合、 **[Customer Detailed Data]\(顧客詳細データ\)** 内の情報はマスクされます (************)。
- Customer ディメンションの詳細 (会社名、顧客名、顧客の電子メールなど) は、Azure Marketplace や AppSource トランザクションのレベルではなく、組織 ID のレベルにあります。

### <a name="customer-page-filters"></a>顧客ページのフィルター

**[顧客] ページ**のフィルターは、[顧客] ページ レベルで適用されます。 ユーザーは、複数のフィルターを選択して、選択した表示条件に合うグラフや、[Detailed orders data] (詳細な注文データ) グリッド/エクスポートに表示したいデータを表示できます。 フィルターは、[orders]\(注文\) ページの右上隅で選択したデータ範囲に対して抽出されたデータに適用されます。

>[!NOTE]
> [Customer] (顧客) グリッドの各フィールドの詳細な定義、ページ フィルター、およびそれらで使用可能な選択については、以下のよくあるご質問についての記事で説明しています。
