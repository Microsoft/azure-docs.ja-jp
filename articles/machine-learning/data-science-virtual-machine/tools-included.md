# <a name="what-tools-are-included-on-the-azure-data-science-virtual-machine"></a>Azure Data Science Virtual Machine に含まれるツール

Data Science Virtual Machine は、クラウドでデータを探索し、機械学習を実行するための簡単な方法です。 Data Science Virtual Machine は、完全なオペレーティング システム、セキュリティ パッチ、ドライバー、一般的なデータ サイエンスおよび開発ソフトウェアを使用してあらかじめ構成されています。 低コストの CPU 中心のマシンから、複数の GPU、NVMe ストレージ、および大容量メモリを搭載した非常に強力なマシンに至るまで、ハードウェア環境を選択することができます。 GPU を搭載したマシンでは、すべてのドライバーがインストールされ、すべての機械学習フレームワークのバージョンが GPU の互換性と一致しています。また、GPU をサポートするすべてのアプリケーション ソフトウェアでアクセラレーションが有効化されています。

Data Science Virtual Machine では、最も有用なデータ サイエンス ツールがプレインストールされます。 

> [!NOTE]
> 次の VM イメージは、どちらも非推奨とされています。
>
> * Windows Server 2016 DSVM イメージ
> * Ubuntu 16.04 DSVM イメージ 
>
> これらの VM イメージは、ARM デプロイを使用している既存のお客様のみが利用できます。 これらのイメージは、提供終了日までは重要なセキュリティ パッチを引き続き受信されますが、今後の最新のデータ サイエンス ツールでは更新されません。
> 
> [DSVM イメージの非推奨化に関するページ](./reference-deprecation.md)を参照してください。

## <a name="how-to-choosecondaenvironment"></a>Conda 環境の選択方法  

> [!NOTE]
> Data Science Virtual Machine の以前のバージョンから新しいバージョンに切り替える場合は重大な変更があります。 

**Windows Server 2016 から Windows Server 2019 への切り替え** </br> 

* Conda 環境は python37_default に設定されています。 python2 はサポートされていません。 python conda 環境で TensorFlow または pytorch を使用していた場合は、それぞれ py37_tensorflow または py37_pytorch を使用します。 AzureML のみを使用していた場合は、azureml_py36_automl を使用します。 </br>

* AzureML Conda 環境で TensorFlow または pytorch を使用していた場合は、それぞれ azureml_py36_tensorflow または azureml_py36_pytorch を使用します。 </br>

**Ubuntu 16.04 から Ubuntu 18.04 への切り替え** </br>

* azureml_py36 Conda 環境で TensorFlow または pytorch を使用していた場合は、それぞれ azureml_py36_tensorflow または azureml_py36_pytorch を使用します。 </br>

* py37 Conda 環境で TensorFlow または pytorch を使用していた場合は、それぞれ py37_tensorflow または py37_pytorch を使用します。 </br>

* それ以外の場合は python37_default を使用するか、azureml_py36 のみを使用していた場合は azureml_py36_automl を使用します。 </br>
