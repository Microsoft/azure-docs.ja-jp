# <a name="choosing-the-right-pricing-tier-in-azure-maps"></a>Azure Maps での適切な価格レベルの選択

Azure Maps では 2 つの価格レベルが提供されています。 この記事は、ニーズにあった適切な価格レベルを選択するのに役立ちます。 適切な価格レベルを選択するには、次の 2 つの質問に答えてください。

## <a name="what-geospatial-capabilities-do-i-plan-to-use"></a>どのような地理空間機能を使用する予定ですか
自分のサービス要件がコア地理空間 API によって満たされると思われる場合は、S0 価格レベルが適しています。 ハイブリッド航空映像、等時線の取得、バッチ ジオコーディングなどのさらに高度な機能がアプリケーションに必要な場合は、S1 価格レベルを検討します。 後で示す**価格レベルの機能**に関する表では、アプリケーションのニーズとそれに最も適した価格レベルの選択に役立つ情報を提供します。

## <a name="how-many-concurrent-users-do-i-plan-to-support"></a>サポートする同時実行ユーザーの数はどれくらいですか 
S0 価格レベルと S1 価格レベルでは、処理できるデータ スループットの量が異なります。 Azure Maps の価格レベルを選択する前に、サポートする予定の同時実行ユーザーの数を検討してください。 S0 価格レベルで処理できるのは最大 **50 クエリ/秒**ですが、S1 価格レベルでは **50 クエリ/秒より多く**を処理できます。

### <a name="pricing-tier-capabilities"></a>価格レベルの機能

| 機能                              |        S0           |  S1      |
|-----------------------------------------|:-------------------:|:--------:|
| Search                                  |        ✓           |     ✓    |
| ルーティング                                 |        ✓           |     ✓    |
| レンダー                                  |        ✓           |     ✓    |
| トラフィック                                 |        ✓           |     ✓    |
| タイム ゾーン                              |        ✓           |     ✓    |
| * 航空映像 + ハイブリッド航空映像 (プレビュー)     |        ✓           |     ✓    |
| * 等時線 (プレビュー)                  |        ✓           |     ✓    |
| * IP 2 の場所 (プレビュー)                |        ✓           |     ✓    |
| * 検索からの多角形 (プレビュー)         |        ✓           |     ✓    |
| * バッチ ジオコーディング (プレビュー)              |        ✓           |     ✓    |
| * バッチ ルーティング (プレビュー)                |        ✓           |     ✓    |
| * マトリックス ルーティング (プレビュー)               |        ✓           |     ✓    |

> [!Note]
> S0 価格レベルからのこれらの機能へのアクセスは、2019 年 2 月 4 日以降に非推奨になります。

他に検討する価値のあるデータ ポイントは、エンタープライズの種類および作成しているアプリケーションの重大度です。

S0 および S1 価格レベルをさらによく理解するには、「**価格レベルの対象のお客様**」の表をご覧ください。 Azure Maps の価格について詳しくは、「[Azure Maps の価格](https://azure.microsoft.com/pricing/details/azure-maps/)」をご覧ください。 

### <a name="pricing-tier-targeted-customers"></a>価格レベルの対象のお客様

| 価格レベル   |        対象となるお客様                                                                |
|---------------|:-----------------------------------------------------------------------------------------|
| S0            |    <p>S0 価格レベルの対象は、小規模または中規模の企業のお客様です。 同時実行ユーザーの数が多くなく、上の表で示されるコア地理空間 API によってサービスの要件が満たされる場合に適した価格レベルです。 このレベルは一般に使用可能であり、概念実証や早い段階のテストからアプリケーションの運用や展開まで、すべての運用ステージのアプリケーションに適用します。<p>|
| S1            |    <p>S1 価格レベルは、大企業や、ミッションクリティカルなアプリケーション、多数の同時使用ユーザーのサポート、高度な地理空間サービスを必要とするお客様に適しています。</p>|
