# <a name="enable-the-preview-experience-to-administer-your-managed-domain-from-the-azure-portal"></a>プレビュー エクスペリエンスを有効にして、Azure Portal から管理対象ドメインを管理する

## <a name="your-existing-managed-domain-is-not-shown-in-the-azure-portal-preview-experience"></a>既存の管理対象ドメインは、Azure Portal の (プレビュー) エクスペリエンスに表示されない
Azure クラシック ポータルを使用して作成された、既存の [Azure AD Domain Services](active-directory-ds-overview.md) 管理対象ドメインがあるとします。 Azure Portal に移動すると、この既存の管理対象ドメインは Azure AD ドメイン サービスのブレード (プレビュー) に一覧表示されません。

![既存の管理対象ドメインは、Azure Portal には表示されない](./media/getting-started/missing-domain-services-instance.png)

これは、既存の管理対象ドメインが新しい Azure Portal に移行されていないためです。 既存の管理対象ドメインを新しい Azure Portal (プレビュー) エクスペリエンスで有効にするには、お問い合わせください。


## <a name="join-the-preview-for-the-azure-portal-experience-for-azure-ad-domain-services"></a>Azure AD Domain Services の Azure Portal エクスペリエンス プレビューに参加する
[お問い合わせ](active-directory-ds-contact-us.md)ページの指示に従って、ご連絡ください。 Azure Portal プレビューで既存の管理対象ドメインを有効にいたします。

> [!NOTE]
> Azure AD Domain Services の新しい Azure Portal エクスペリエンスが一般公開 (GA) されると、すべての既存の管理対象ドメインは自動的に移行されます。 その時点で、既存の管理対象ドメインを Azure Portal から管理できるようになります。
