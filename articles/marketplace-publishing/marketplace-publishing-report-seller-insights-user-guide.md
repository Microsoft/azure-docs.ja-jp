# <a name="how-to-use-the-microsoft-azure-marketplace-seller-insights-portal"></a>Microsoft Azure Marketplace の Seller Insights Portal を使用する方法

##<a name="signing-in"></a>サインイン

管理者からの電子メールで、販売者のアカウントが作成されたことと、Seller Insights Portal の URL (https://reports.azure.com) が通知されます。

Seller Insights Portal にログインするには、個人の Microsoft アカウントを使用するか、会社用のアカウントを使用します。これは、管理者がプロフィールの設定にどちらのアカウントを使用したかによって決まります。  管理者が使用するアカウントを知らせるはずです。

   ![type of account][1]



"***Seller Insights Portal へようこそ***"

1. ログインすると、[Summary (概要)] ページが表示されます。
2. 少し時間を取ってプライバシー ポリシーを読んでください。
3. Seller Insights Portal に関する質問はいつでも電子メールでお問い合わせいただけます。
4. Seller Insights Portal 内では、ページの上部にあるタブを使用して移動できます。  このガイドでもページ上部のタブを見ると、Seller Insights Portal 内の場所を確認できます。
5. いつでもスマイル マークをクリックして、Seller Insights Portal に関連するフィードバックを送信できます。
6. いつでも Seller Insights Portal からサインアウトするには、右上にある自分の名前をクリックします。

   ![default sign in page][2]



##<a name="resetting-a-password"></a>パスワードのリセット

"***アカウントとパスワードの検証***"

1. "***使用しているアカウントが登録されていることと、パスワードが機能しているかどうかを検証します。***" Azure Active Directory に登録されたアカウントに入ると、そのアカウントのログイン方法を選択するように自動的に求められます。つまり、職場または学校 (組織アカウント) または個人の LiveID アカウントを選択します。
2. https://reports.azure.com に移動して、ログインに使用した可能性があるアカウントからログアウトするか、プライベート ウィンドウで https://reports.azure.com を開いて、他のアカウントでログインしていないことを確認します。
3. 発行者の電子メール アドレスを入力します。パスワードは入力しません。 このアドレスが Azure Active Directory に登録されている場合は、***手順 4*** に進みます (管理者アカウントは登録されています。管理者アカウントによって設定されるすべての組織アカウントも同様です)。
4. [職場または学校アカウント] を選択します。 職場または学校アカウント (組織アカウント) は Azure に登録されている組織によって設定されます。個人アカウント (LiveID アカウント) は Microsoft に登録されている個人用アカウントです。
 * ユーザーが以前に自分の会社の電子メール アドレスを使用して LiveID アカウントを設定し、その後で会社が Azure の使用を開始した場合、同じ電子メール アドレスが組織アカウントとしても登録されることが可能です。 つまり、このユーザーは、LiveID を作成するために使用したパスワードが設定された LiveID アカウントと、会社が使用する別のパスワードが設定された組織アカウントを持つことになります。
5. Seller Insights Portal で [職場または学校アカウント] を選択するときは、LiveID とパスワードの組み合わせではなく、組織アカウントとパスワードの組み合わせを使用してください。
6. 組織アカウントの場合、発行者のログイン ページにリダイレクトされます。
7. ここで、組織アカウントのパスワードが機能しているかどうかを検証します。 機能しない場合は、"***パスワードのリセット***" に進みます。

"***パスワードのリセット***"

後で示すリンクで、[セルフサービスによるパスワードのリセット] ウィザードを使用してパスワードをリセットします。
 * 注: 管理者アカウントによって設定された組織アカウントでは、管理者がユーザーが自分のパスワードをリセットできる機能を有効にした場合、ユーザーが自分のパスワードをリセットできます。 それ以外の場合は、管理者に連絡するためのオプションがパスワード リセット フォームに表示されます。
