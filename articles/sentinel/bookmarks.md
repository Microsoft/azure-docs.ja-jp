# <a name="keep-track-of-data-during-hunting"></a>検出中のデータの追跡

> [!IMPORTANT]
> 現在、Azure Sentinel はパブリック プレビュー段階にあります。
> このプレビュー バージョンはサービス レベル アグリーメントなしで提供されています。運用環境のワークロードに使用することはお勧めできません。 特定の機能はサポート対象ではなく、機能が制限されることがあります。 詳しくは、[Microsoft Azure プレビューの追加使用条件](https://azure.microsoft.com/support/legal/preview-supplemental-terms/)に関するページをご覧ください。
 
脅威検出では、通常、山のようなログ データを調べて、悪意のある動作の証拠を探す必要があります。 このプロセス中に調査担当者は、可能性のある仮説の実証と、侵害の詳細の把握の一環として、記憶し、再考し、分析する必要のあるイベントを探します。
検出ブックマークは、Log Analytics で実行したクエリを、関連すると思われたクエリ結果とともに保持することにより、これらの操作に役立ちます。 また、メモやタグを追加することにより、コンテキストに関する所見を記録し、調査結果を参照することもできます。 ブックマークが設定されたデータは、自身とチームメイトが見ることができコラボレーションを簡単に行えるようにしています。   

**[Hunting]** (検出) ページの **[ブックマーク]** タブで、ブックマークが設定されたデータにいつでも再訪できます。 フィルタリングおよび検索オプションを使用して、現在の調査の特定のデータをすばやく見つけられます。 または、ブックマークが設定されたデータを、Log Analytics の **[HuntingBookmark]** (検出ブックマーク) テーブルで直接確認できます。 これにより、ブックマークが設定されたデータをフィルター処理、要約、および他のデータ ソースと結合でき、対応する証拠を簡単に見つけられます。

**[調査]** をクリックすることにより、ブックマークが設定されたデータを視覚化することもできます。 これにより、対話型のエンティティ-グラフ ダイアグラムおよびタイムラインを使用して、調査結果を表示、調査、および視覚的に伝達できる調査エクスペリエンスが開始します。


## <a name="run-a-log-analytics-query-from-azure-sentinel"></a>Azure Sentinel から Log Analytics クエリを実行する

1. Azure Sentinel ポータルで、 **[Hunting]** (検出) をクリックして、不審および異常な動作についてクエリを実行します。

1. 検出キャンペーンを実行するには、検出クエリのいずれかを選択し、左側で結果を確認します。 

1. 検出クエリの **[詳細]** ページの **[View query results]** (クエリ結果の表示) をクリックして、Log Analytics でクエリ結果を表示します。 カスタム SSH 総当たり攻撃クエリを実行した場合の表示例を次に示します。

4. 行の末尾にある省略記号 (...) を選択し、 **[Add hunting bookmarks]** (検出ブックマークの追加) を選択します。
5. 右側の **[詳細]** ページで、名前を更新し、タグを追加し、項目に関して興味深かったことを識別するために役立つメモを追加します。
6. **[保存]** をクリックして変更を確定します。 ブックマークが設定されたすべてのデータは他の調査担当者と共有され、共同の調査エクスペリエンスに向けた最初の手順になります。

> [!NOTE]
> [Azure Sentinel Log Analytics Logs]\(Azure Sentinel Log Analytics ログ) ページから開始された恣意的な Log Analytics クエリでブックマークを使用することも、[Log Analytics] ページから進行中に作成され、[Hunting]\(検出) ページから開かれたクエリでブックマークを使用することもできます。 Azure Sentinel の外部から Log Analytics を起動した場合、ブックマークを追加することはできません。 

## <a name="view-and-update-bookmarks"></a>ブックマークを表示および更新する 

1. Azure Sentinel ポータルで、 **[Hunting]** (検出) をクリックします。 
2. ページの中央にある **[ブックマーク]** タブをクリックして、ブックマークの一覧を表示します。
3. 検索ボックスやフィルター オプションを使用して、特定のブックマークを検索します。
4. 下のグリッドの個々のブックマークを選択して、右側の詳細ウィンドウでブックマークの詳細を表示します。
5. タグとメモを更新するには、編集可能なテキスト ボックスをクリックし、 **[保存]** をクリックして変更を保持します。

   ![結果の表示](./media/bookmarks/view-update-bookmarks.png)

## <a name="view-bookmarked-data-in-log-analytics"></a>Log Analytics でブックマークが設定されたデータを表示する 

Log Analytics でブックマークが設定されたデータを表示する複数のオプションがあります。 

ブックマークが設定されたクエリ、結果、または履歴を表示する最も簡単な方法は、 **[ブックマーク]** テーブルで目的のブックマークを選択し、詳細ウィンドウで表示されるリンクを使用します。 次のオプションがあります。 
- **[クエリを表示]** をクリックして、Log Analytics でソース クエリを表示します。  
- **[View bookmark history]** (ブックマーク履歴の表示) をクリックして、更新を行ったユーザー、更新された値、更新が行われた時刻など、すべてのブックマーク メタデータを表示します。 

- ブックマーク グリッドの上の **[Bookmark log]** (ブックマーク ログ) をクリックして、すべてのブックマークの生ブックマーク データを表示することもできます。 このビューには、関連付けられているメタデータを持つ検出ブックマーク テーブルのすべてのブックマークが表示されます。 KQL クエリを使用して、検索している特定のブックマークの最新版に絞り込むことができます。  


> [!NOTE]
> ブックマークの作成と、それが **[HuntingBookmark]** (検出ブックマーク) テーブルに表示されるときとの間に、著しい遅れ (分数で測定) が発生することがあります。 最初にブックマークを作成してから、データが取り込まれた後に分析することをお勧めします。 

## <a name="delete-a-bookmark"></a>ブックマークを削除する
ブックマークを削除する場合、次の手順に従います。 
1.  **[Hunting bookmark]** (検出ブックマーク) タブを開きます。 
2.  ターゲットのブックマークを選択します。
3.  行の末尾にある省略記号 (...) を選択し、 **[ブックマークを削除]** を選択します。
    
ブックマークを削除すると、 **[ブックマーク]** タブの一覧からブックマークが除かれます。Log Analytics の「HuntingBookmark」テーブルには引き続き、以前のブックマーク エントリが含まれますが、最新のエントリは **[SoftDelete]** 値を true に変更され、この結果、古いブックマークの除外が容易になります。  ブックマークを削除しても、他のブックマークやアラートに関連付けられている調査エクスペリエンスからはどのエンティティも削除されません。
